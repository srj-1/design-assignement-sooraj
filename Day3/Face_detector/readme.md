# Task 2: Design and Integration of Face Detection Module, FIFO Buffer, and FSM-Based Output Controller

## 1. Objective

To design and integrate a digital system consisting of a Face Detection Module, an 8-bit FIFO Buffer, and an FSM-Based Output Controller using Verilog HDL. The system is verified through simulation in Xilinx Vivado to ensure proper data acquisition, buffering, and controlled output generation.

---

# 2. Introduction

In digital systems, modules often operate at different data rates. When one module generates data faster than another module can process it, a buffering mechanism is required to prevent data loss.

In this project, a three-stage architecture was implemented:

1. Face Detection Module (Face_Detect_Mod)
2. FIFO Buffer
3. FSM-Based Output Module (mod_out)

The Face Detection Module captures incoming 8-bit data every clock cycle. Since the Output Module produces results only once every three clock cycles, a FIFO buffer is introduced between the two modules to temporarily store data and maintain synchronization.

---

# 3. Problem Statement

The Face Detection Module produces valid data on every positive edge of the clock, while the Output Module is designed to generate output only on every third positive clock edge.

### Data Rate Mismatch

| Module          | Data Rate                 |
| --------------- | ------------------------- |
| Face_Detect_Mod | 1 data per clock cycle    |
| mod_out         | 1 data per 3 clock cycles |

Because the producer is faster than the consumer, direct connection of the modules would result in data loss. Therefore, a FIFO buffer is required to temporarily store incoming data until the Output Module is ready to process it.

---

# 4. System Architecture

The complete system consists of three interconnected modules.

```text
           s_in[7:0]
                │
                ▼
      +------------------+
      | Face_Detect_Mod  |
      +------------------+
                │
                ▼
      +------------------+
      |      FIFO        |
      | 8-bit Buffer     |
      +------------------+
                │
                ▼
      +------------------+
      |     mod_out      |
      | FSM Controller   |
      +------------------+
                │
                ▼
           d_out[7:0]
```

---

# 5. Module Description

## 5.1 Face_Detect_Mod

### Purpose

Acts as the data acquisition stage of the system.

### Inputs

* clk
* s_in[7:0]

### Output

* s_out[7:0]

### Functionality

The module captures incoming data at every positive edge of the clock and forwards it to the FIFO.

### Operation

```verilog
always @(posedge clk)
begin
    s_out <= s_in;
end
```

### Features

* 8-bit input and output
* Synchronous operation
* Data registered on every clock cycle

---

## 5.2 FIFO Buffer

### Purpose

Provides temporary storage between the producer and consumer modules.

### Inputs

* clk
* rst
* data_in[7:0]
* wrenb
* rdenb

### Outputs

* data_out[7:0]

### Features

* 8-bit data width
* FIFO (First-In-First-Out) operation
* Synchronous read and write
* Temporary buffering of incoming data

### Working Principle

Data generated by Face_Detect_Mod is continuously written into the FIFO. When requested, the FIFO supplies stored data to the Output Module in the same order in which it was received.

---

## 5.3 mod_out

### Purpose

Controls the timing of output generation using a Finite State Machine (FSM).

### Inputs

* clk
* rst
* d_in[7:0]

### Outputs

* d_out[7:0]

### FSM States

| State | Function                         |
| ----- | -------------------------------- |
| IDLE  | Capture FIFO data                |
| S1    | Wait cycle 1                     |
| S2    | Wait cycle 2 and generate output |

### State Transition Diagram

```text
IDLE → S1 → S2 → IDLE
```

### Working Principle

1. Data from FIFO is captured in the IDLE state.
2. The FSM waits for two additional clock cycles.
3. During state S2, the stored data is transferred to the output.
4. The process repeats for subsequent data.

Thus, one output is generated for every three positive clock edges.

### Step 2

Design and verify the FIFO Buffer.

### Step 3

Design and verify the FSM-based Output Module.

### Step 4

Integrate all three modules into a single top-level design.

---

# 7. Top Module Integration

The top-level module connects all three blocks.

### Data Flow

1. External input data enters Face_Detect_Mod.
2. Captured data is forwarded to FIFO.
3. FIFO stores incoming values.
4. FIFO output is connected to mod_out.
5. mod_out generates the final output every third clock cycle.

This architecture ensures reliable data transfer despite different processing rates.
